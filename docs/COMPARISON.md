# iOS App vs Web Dashboard 对比

Mac Monitor 现在提供两种使用方式，每种都有其优势。本文档帮助你选择最适合的版本。

## 快速对比

| 功能特性 | iOS 原生 App | Web Dashboard |
|---------|-------------|---------------|
| 🎨 **界面** | SwiftUI 原生 | HTML/CSS/JS |
| 📱 **运行平台** | iOS 16+ | 任何浏览器 |
| 🚀 **性能** | 原生，最佳 | 良好 |
| 💾 **安装** | Xcode编译安装 | 无需安装 |
| 🔔 **推送通知** | ✅ 本地推送 | ❌ 不支持 |
| 🌓 **暗色模式** | ✅ 系统集成 | ✅ 手动切换 |
| 🔍 **设备发现** | Network Framework | JavaScript扫描 |
| 🔄 **自动刷新** | ✅ 5秒 | ✅ 5秒 |
| 💿 **数据持久化** | UserDefaults | localStorage |
| 📊 **监控卡片** | 6个 | 6个 |
| 🎯 **横屏优化** | ✅ 自适应 | ✅ 响应式 |

## 使用建议

### 推荐使用 iOS 原生 App 的场景：
- **主力使用设备是 iPhone** - 获得最佳的原生体验
- **需要推送通知提醒** - CPU/内存告警时自动推送
- **长期监控使用** - 原生性能更好，电池优化更佳
- **有 Xcode 和开发环境** - 方便编译和安装

### 推荐使用 Web Dashboard 的场景：
- **快速测试或演示** - 无需安装，打开浏览器即用
- **没有 Xcode 环境** - Linux、Windows 用户
- **多设备监控** - iPad、Mac、Android 都能用
- **临时使用** - 不想安装 app

## 总结

**选择建议：**
- 🎯 **主力用户** → iOS 原生 App
- 🚀 **快速测试** → Web Dashboard
- 💼 **团队使用** → Web Dashboard
- 📱 **个人监控** → iOS 原生 App

两个版本都连接同一个 Python Agent 后端，数据完全一致。你可以同时使用两者，根据场景选择！
